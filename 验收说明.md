# iPhone 16 离线 Demo 验收说明

> 本文件只验收当前离线 Demo。项目级产品需求、信息架构与正式落地边界以 `PRD.md` 为准；Demo 详细规格见 `demo/原型说明.md`。

## 环境

- 逻辑视口：393×852。
- Device Scale Factor：3。
- 触控模式：开启。
- 输出截图：1179×2556。
- 运行协议：`file://`。

## 一键验收

在项目根目录运行：

```powershell
node scripts/verify_demo.mjs
```

脚本验证：

- 17个 PNG 文件名和 SHA-256 与参考项目一致。
- 主画板尺寸和截图像素尺寸正确。
- 15个讲解点图标映射正确。
- 便民服务为2个服务点加3个WC点。
- 无障碍服务为3个WC点加5个坡板点。
- 定位授权、最近讲解、媒体模式、下一步、服务详情和语音指引可用。
- 从详情返回时恢复筛选、选中状态和焦点。
- 页面无远程网络请求、控制台错误和320px横向溢出。

## 项目级 PRD 交付验收

```powershell
node scripts/verify_prd_html.mjs
python scripts/verify_prd_outputs.py
```

脚本验证：

- 主 PRD 包含项目背景、用户需求、三层信息架构、产品需求、交付状态、责任、里程碑和待确认事项。
- 固定推荐顺序等 Demo 细节只存在于 `demo/原型说明.md`。
- HTML 的项目看板、内容域卡片、信息架构树、任务流程、状态标签和里程碑均正常渲染。
- HTML 在1440、768、393和320px视口无页面级横向溢出，且不加载远程运行资源。
- PDF 使用稳定文件名生成，全部页面由 Poppler 渲染到 `tmp/pdfs/` 供逐页视觉检查。

## 参考项目对照

首次运行参考项目：

```powershell
cd "C:\Users\10189\Downloads\无障碍在线导览界面设计"
npm install --no-package-lock --ignore-scripts --no-audit --no-fund
npm run dev -- --host 127.0.0.1
```


## 人工检查

1. 双击 `demo/index.html`，确认无需服务器即可进入地图。
2. 检查393×852下没有文字截断，底部按钮不被 Home Indicator 遮挡。
3. 检查320px下图例和按钮不横向溢出。
4. 检查桌面环境中只有一个393px宽的居中界面。
5. 使用键盘切换筛选、点位、弹层和返回按钮，确认焦点可见。
6. 使用读屏确认点位名称同时包含地点与服务类型。
